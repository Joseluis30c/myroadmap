Dependency Injection Profundo
Lifetimes, Scoped vs Singleton vs Transient, y patrones avanzados
⏱ ~5h
Concepto clave: DI en .NET tiene 3 lifetimes: Transient (nueva instancia cada vez), Scoped (una por request HTTP) y Singleton (una por toda la vida de la app). Mezclarlos mal provoca bugs sutiles y memory leaks (Captive Dependencies).

Configuración Avanzada
appsettings, environments, Options Pattern, secrets y variables de entorno
⏱ ~4h
Concepto clave: ASP.NET Core usa un sistema de configuración por capas: appsettings.json → appsettings.{{'{'}}Env{{'}'}}.json → Variables de entorno → Argumentos CLI → User Secrets. Cada capa puede sobrescribir la anterior. El Options Pattern (IOptions, IOptionsSnapshot, IOptionsMonitor) es la forma correcta de consumirla.

🎯 Entregable Final – API Bien Estructurada

Tu API debe cumplir todos estos puntos para considerar la semana completa:

  • Middleware personalizado de timing y manejo de errores
  • Servicios registrados con el lifetime correcto (Scoped para repositorios, etc.)
  • Configuración tipada con Options Pattern y validación
  • appsettings separado por environment (Development / Production)
  • Serilog configurado desde appsettings.json con structured logging
  • Endpoints mínimamente estructurados (al menos CRUD de un recurso)
  • Código limpio: sin lógica de negocio en Program.cs
